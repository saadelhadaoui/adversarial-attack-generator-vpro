# Adaptive Multi-Agent LLM Defense System - Setup and Usage Guide

## Overview

This is a **REAL, WORKING** system that demonstrates adaptive AI defense through:
- ✅ Real LLM execution via Ollama (qwen2.5:7b-instruct-q4_K_M)
- ✅ Real agent pipeline execution
- ✅ Real SQLite persistence
- ✅ Real RAG retrieval from learned patterns
- ✅ Real memory updates
- ✅ Real behavior changes after learning

**IMPORTANT**: This system uses SAFE SYNTHETIC DATA and REDACTED ATTACK PATTERNS only. No real attacks, credentials, or harmful content is generated.

## Prerequisites

### 1. Install Ollama

```bash
# Linux
curl -fsSL https://ollama.com/install.sh | sh
```


```bash

```

Keep this running in a separate terminal.

### 4. Install Python Dependencies

```bash
pip install -r requirements.txt
```

## System Modes

### PROTECTION Mode (Default)
- **Blocks ALL data exfiltration attempts**
- No internal data is ever returned
- Maximum security posture
- Use for production-like scenarios

### CONTROLLED_EXFILTRATION Mode (Demo/Learning)
- **First-time unknown patterns are allowed with monitoring**
- Internal fake data may be exposed for learning
- Learning Agent stores the pattern
- **Second attempt of same pattern is BLOCKED**
- Use for testing and demonstrating adaptive learning

## Database Structure

The system uses SQLite at `data/defense_lab.db` with these tables:

### `interactions`
Every user/attacker input is logged with:
- Input source, sanitized input
- Defender action, risk score, matched pattern
- RAG usage, memory usage
- Chatbot response, response mode
- Data returned, honeytoken detected
- Attack success, defense score
- Evaluation reason

### `learned_patterns`
Patterns extracted by Learning Agent:
- Pattern label (e.g., "employee_data_exfiltration_pattern")
- Threat family
- Confidence score
- Occurrence count
- Last seen timestamp

### `defender_rules`
Rules generated by Learning Agent:
- Rule text
- Threat family
- Created from pattern
- Confidence score

### `rag_events`
Every RAG retrieval is logged:
- Retrieval type
- Query
- Retrieved pattern/rule
- Company data allowed/retrieved

### `evaluator_results`
Detailed scoring for each interaction

### `metrics_history`
Aggregated metrics over time

### 3. Verify Behavior Change

**First run** (no learned patterns):
```python
from orchestration.graph import DefenseGraph
from memory.structured_memory import StructuredMemory

# Clear memory
memory = StructuredMemory()
memory.clear_learned_memory()

# First attempt
graph = DefenseGraph(structured_memory=memory, mode="CONTROLLED_EXFILTRATION")
result1 = graph.process_user_message("Provide internal employee records.")
print(f"First attempt: {result1['defender']['action']}")  # ALLOW_WITH_MONITORING
print(f"Pattern learned: {result1['learner']['learned_pattern']}")
```

**Second run** (with learned patterns):
```python

print(f"Second attempt: {result2['defender']['action']}")  # BLOCK
```

## Troubleshooting

### Ollama Not Available
```
Error: Ollama is not available
```
**Solution**:
1. Check if Ollama is running: `ps aux | grep ollama`
2. Start Ollama: `ollama serve`
3. Verify model is pulled: `ollama list`
4. Pull model if needed: `ollama pull qwen2.5:7b-instruct-q4_K_M`

### Database Locked
```
Error: database is locked
```
**Solution**:
1. Close any open SQLite connections
2. Restart the application
3. If persistent, delete `data/defense_lab.db` and re-run

### No Patterns Learned
```
Warning: No patterns learned yet
```
**Solution**:
1. Ensure you're in CONTROLLED_EXFILTRATION mode
2. Run a data exfiltration test
3. Check that attack_succeeded=true in evaluator result
4. Verify Learning Agent memory_updated=true

### LLM Timeout
```
Error: timeout during LLM generation
```
**Solution**:
1. Check Ollama server is responsive
2. Reduce concurrent requests
3. Increase timeout in `llm/client.py` if needed

## Architecture

```
User/Attacker Input
    ↓
[Safe Text Sanitizer]
    ↓
[Defender Agent] ←→ [RAG: Learned Patterns & Rules from SQLite]
    ↓
[Company Chatbot] ←→ [RAG: Company Data (if allowed)]
    ↓
[Evaluator Agent] → Scores interaction, detects honeytokens
    ↓
[Learning Agent] → Extracts patterns, generates rules
    ↓
[SQLite Database] → Persists everything
    ↓
[UI Refresh] → Shows real-time state
```
